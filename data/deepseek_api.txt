DeepSeek API 提供了简洁的 HTTP 接口，开发者可以通过标准 RESTful API 调用 DeepSeek 系列模型。API 完全兼容 OpenAI 的接口格式，因此可以无缝迁移现有的 OpenAI 调用代码。

**API 调用基础**
DeepSeek API 的 base_url 为 https://api.deepseek.com/v1。开发者需要在请求头中携带 API Key 进行身份认证，API Key 可以在 DeepSeek 开放平台的控制台中生成和管理。

**支持的模型**
目前 DeepSeek API 支持以下模型：
- deepseek-chat (DeepSeek-V3) - 通用对话模型，适合各种任务
- deepseek-reasoner (DeepSeek-R1) - 推理增强模型，适合复杂推理任务

**核心功能**
1. 对话补全（Chat Completion）：支持多轮对话，可以设置 system message 和 user message。
2. 流式输出（Streaming）：通过设置 stream=true 实现 token 级别的实时输出。
3. 函数调用（Function Calling）：模型可以根据需要调用你定义的函数。
4. 上下文缓存（Context Caching）：对频繁使用的上下文进行缓存，降低延迟和成本。
5. FIM 补全（Fill-in-the-Middle）：适合代码补全场景。

**参数说明**
- temperature：控制输出的随机性，范围 0-1，默认 0.7。
- max_tokens：限制生成的最大 token 数量。
- top_p：核采样参数，默认 1。
- frequency_penalty：频率惩罚，减少重复内容。
- presence_penalty：存在惩罚，鼓励讨论新话题。

**注意事项**
DeepSeek 不支持 function calling 外部图片、外部图片生成、以及 embeddings 接口。所有 API 调用均按 token 计费，输入和输出分别计费。
