以下是一些使用 DeepSeek 模型的最佳实践，帮助你获得更好的结果：

**1. 编写清晰的 Prompt**
- 使用明确、具体的指令，而不是模糊的描述。
- 为模型指定角色（如"你是一名Python专家"）可以提高回答质量。
- 将复杂任务分解为多个步骤，引导模型逐步完成。

**2. 利用 System Message**
System message 是设定模型行为的重要工具。你可以用它来定义输出格式、约束行为范围、指定专业领域等。例如：system message 可以规定"请用中文回答，保持专业简洁的语调"。

**3. 善用 Few-shot 示例**
在 prompt 中提供 1-3 个示例可以显著提升模型在特定任务上的表现。示例应覆盖任务的主要变体，包括输入和期望的输出格式。

**4. 温度参数调优**
- 对于需要确定答案的任务（如代码生成、事实查询），推荐使用较低的 temperature（0-0.3）。
- 对于创意性任务（如写作、头脑风暴），可以使用稍高的 temperature（0.7-0.9）。
- 默认值 0.7 在大多数场景下表现均衡。

**5. 利用长上下文**
DeepSeek 支持 128K 上下文，你可以在 prompt 中提供完整的参考材料，让模型基于你提供的信息进行回答。这种方法比 RAG 更直接，但成本也更高。对于长文档分析，建议将关键信息放在 prompt 的开头和结尾部分。

**6. 函数调用最佳实践**
- 函数定义应清晰简洁，包含详细的参数描述。
- 避免定义过多的函数，一次调用中建议不超过 10 个工具。
- 使用 strict=true 可以确保模型严格按照函数定义输出。

**7. 错误处理**
API 调用可能因各种原因失败，建议实现重试机制和退避策略。常见的错误包括速率限制、认证失败、服务暂不可用等。

**8. 成本优化**
- 使用上下文缓存减少重复输入的 token 消耗。
- 对于简单任务，精简 prompt 长度可以显著降低成本。
- 考虑使用流式输出提升用户体验，而不是等待完整响应。
